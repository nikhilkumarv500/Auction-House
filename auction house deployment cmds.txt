Backend


1.)do
./mvnw clean package -DskipTests
if it gives error for no env varialbes found

2.)set env variables in command line
powershell =>
$env:SPRING_APPLICATION_NAME=""

windows (command prompt)
set SPRING_APPLICATION_NAME=""

3.) again do step 1

4.) create docker file

5.) in IntelliJ powershell do
docker build -t demo-deployment .

6.) shld tag image with docker hub username, do in powershell
docker tag demo-deployment decode007/demo-deployment:latest

7.) do
docker push decode007/demo-deployment:latest

8.) then in render service
choose a new webservice
then paste link for existing image
url = docker.io/{imageurl}:latest

doing service registory

./mvnw clean package -DskipTests

//the create docker file (Dockerfile)

docker build -t auction-house-serviceregistory .

docker tag auction-house-serviceregistory nikhilkumarv/auction-house-serviceregistory:latest

//doing render push - done
docker.io/nikhilkumarv/auction-house-serviceregistory:latest

//update render environment
